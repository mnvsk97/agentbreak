Minimal chaos testing for LLM apps

Break the provider. Not your app.

AgentBreak is a small proxy for OpenAI-compatible apps. Put it between your app and the model provider, then simulate slow responses, rate limits, and server errors to see whether retries, fallbacks, and timeout handling actually work.

Mock mode

Return fake responses and injected failures locally.

Proxy mode

Forward to the real provider and randomly inject failures into some requests.

AgentBreak -> provider

Even major providers have incidents. OpenAI Status and Claude Status are good reminders that resilience testing matters even more when you run your own stack.

Quick start

Run it locally in a minute

Start in mock mode, point your app at localhost, and test failure handling without changing application code. 

pip install agentbreak

agentbreak start \
  --mode mock \
  --scenario mixed-transient \
  --fail-rate 0.2

export OPENAI_BASE_URL=http://localhost:5000/v1
export OPENAI_API_KEY=dummy
