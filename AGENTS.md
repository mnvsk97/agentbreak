# Agent Guidelines

## Project

AgentBreak is a chaos proxy for:

- OpenAI-compatible `POST /v1/chat/completions`
- MCP over streamable HTTP at `POST /mcp`

Keep the surface small and explicit: typed YAML config, reproducible scenarios, deterministic `agentbreak verify`.

## Layout

| Path | Role |
|------|------|
| `agentbreak/main.py` | CLI (`serve`, `validate`, `inspect`, `verify`), FastAPI app, proxy logic |
| `agentbreak/config.py` | `application.yaml` models, registry load/save |
| `agentbreak/scenarios.py` | `scenarios.yaml` schema and validation |
| `agentbreak/behaviors.py` | Response mutation helpers |
| `agentbreak/discovery/mcp.py` | MCP inspection and parsing |
| `tests/` | Pytest suite; `agentbreak verify` runs pytest |
| `examples/` | LangChain, LangGraph, MCP servers, live harness |
| `docs/` | Failure modes, deferred scenarios, live testing notes |
| `skills/` | Claude Code skills (`agentbreak-testing`, `tester`, `test-generator`) |

## Config

- `application.yaml` -- LLM/MCP settings, upstreams, auth, host/port. Copy from `config.example.yaml`.
- `scenarios.yaml` -- Faults, targets (`llm_chat`, `mcp_tool`), schedules. Copy from `scenarios.example.yaml`.
- `.agentbreak/registry.json` -- Generated by `agentbreak inspect` when MCP is enabled.

There is no `agentbreak start` or legacy `config.yaml` path.

## Commands

```bash
pip install -e '.[dev]'

agentbreak validate --config application.yaml --scenarios scenarios.yaml
agentbreak serve   --config application.yaml --scenarios scenarios.yaml
agentbreak inspect --config application.yaml   # MCP only; writes registry

agentbreak verify        # pytest
agentbreak verify --live # pytest + LangGraph live harness
```

## When Changing Behavior

- Update `scenarios.example.yaml` / `config.example.yaml` if shapes change.
- Extend `tests/` and keep `agentbreak verify` green.
- Update `README.md` for user-visible changes; `CONTRIBUTING.md` for dev workflows.
- Keep `skills/agentbreak-testing/SKILL.md` and `.claude-plugin/commands/agentbreak.md` aligned with the CLI.

## Principles

- One clear path over multiple overlapping CLI modes.
- Test isolation; no flaky network in default CI.
- Do not commit secrets, real API keys, or large generated artifacts.
